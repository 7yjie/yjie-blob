    # 1.JavaScript 词法作用域

    作用域：定义变量的区域。

    作用域规定了如何查找变量，也就是确定当前执行代码对变量的访问权限。

    词法作用域也就是静态作用域 ，是在函数定义的时候就确定了作用域

    var scope = "global scope";
function checkscope(){
    var scope = "local scope";
    function f(){
        return scope;
    }
    return f();
}
checkscope();	

    var scope = "global scope";
function checkscope(){
    var scope = "local scope";
    function f(){
        return scope;
    }
    return f;
}
checkscope()();

    结果都是 local scope JavaScript 函数的执行用到了作用域链，这个作用域链是在函数定义的时候创建的。嵌套的函数 f() 定义在这个作用域链里，其中的变量 scope 一定是局部变量，不管何时何地执行函数 f()，这种绑定在执行 f() 时依然有效。

    # 2.JavaScript 执行上下文

    JavaScript在执行的时候是一段一段分析执行。当执行到一段代码的时候会进行响应的准备工作 （执行上下文 execution context）。

    如何划分段？

    可执行代码：

    ​ 全局代码、函数代码、eval。

    执行上下文栈

    // 上下文栈
ECStack = [];




// 首先遇到的就是全局代码 会创建一个全局的执行上下文 global context 压入到ECStack 只有当整个程序执行结束的时候才会将它弹出栈

function fun3() {
    console.log('fun3')
}

function fun2() {
    fun3();
}

function fun1() {
    fun2();
}

fun1();


//接下来就是这段代码 首先遇到的是 fun1 ，创建一个fun1的上下文并压入栈
ECStack.push(<fun1>,functionContext);
//fun1调用了 fun2 创建 fun2 的上下文并入栈
ECStack.push(<fun2> functionContext);
//fun2还调用了fun3！
ECStack.push(<fun3> functionContext);

// fun3 执行完毕
ECStack.pop();
// fun2 执行完毕
ECStack.pop();
// fun1 执行完毕
ECStack.pop();

    # 3.JavaScript变量对象

    每个执行上下文都有三个属性：

    • 变量对象

      变量对象是与执行上下文有关的数据作用域，存储了在上下文定义的变量和函数声明

      全局上下文：

      ​ 在 客户端的JavaScript当中 全局对象就是指window对象，是通过object构造函数实例化的对象，作为全局变量的宿主。在全局上下文当中的变量对象就是全局对象。

      函数上下文：

      ​ 用活动对象(activation object, AO)来表示变量对象。

    • 作用域链

    • this

    执行过程 ：分析和执行

    ​ 进入执行上下文

    ​ 代码执行

    变量对象包括：

    ​ 1.函数的所有形参（如果是函数上下文）

    • 由名称和对应值组成的一个变量对象的属性被创建
    • 没有实参，属性值设为 undefined

    ​ 2.函数声明

    • 由名称和对应值（函数对象(function-object)）组成一个变量对象的属性被创建
    • 如果变量对象已经存在相同名称的属性，则完全替换这个属性

    ​ 3.变量声明

    • 由名称和对应值（undefined）组成一个变量对象的属性被创建；
    • 如果变量名称跟已经声明的形式参数或函数相同，则变量声明不会干扰已经存在的这类属

    # 4.JavaScript作用域链

    当查找变量的时候，会从当前的执行上下文的变量对象当中寻找，如果没有找到就去父级上下文的变量对象当中寻找，直到全局上下文的变量对象也就是全局对象，由多个执行上下文组成的变量对象构成的链表叫做作用域链

    # 函数创建

    函数的作用域在函数定义的时候就决定了

    函数有一个内部属性 [[scope]],当函数创建的时候就会保存所有父变量对象到其中。

    function foo() {
    function bar() {
        ...
    }
};
    
// 函数创建的时候
foo.[[scope]] = [
    globalContext.VO
];
    
bar.[[scope]] = [

    # 函数激活

    当函数激活的时候，进入执行上下文，创建AO后，将活动对象添加到作用链的 最前端。

    这时候执行上下文的作用域命名为 Scope

    Scope = [AO].concat([[Scope]]);

    例子：

    var scope = "global scope";
function checkscope(){
    var scope2 = 'local scope';
    return scope2;
}
checkscope();


// 执行过程如下
// checkscope 被创建 保存其父级作用链的变量对象
checkscope.[[scope]] = [
    globalContext.VO
];
// 执行 checkscope 函数，创建 checkscope 执行上下文 checkscope 执行上下文被压入上下文栈
ECStack = [
    checkscopeContext,
    globalContext
];
// checkscope 不会立即执行 开始准备工作。第一步 复制函数 scope 属性创建作用域链
checkscopeContext = {
    Scope:checkscope.[[scope]]
};
// 第二步：用 arguments 创建活动对象，随后初始化活动对象，加入形参、函数声明、变量声明
checkscopeContext = {
    AO = {
        arguments:{
            length:0
        },
        scope2:undefined
	},
    Scope:checkscope.[[scope]],
};
// 第三步 将活动对象压入到作用链前端
checkscopeContext = {
    AO = {
        arguments:{
            length:0
        },
        scope2:undefined
	},
    Scope:[AO,checkscope.[[scope]],
};
// 准备工作完成 开始执行函数 修改AO的属性值
checkscopeContext = {
    AO: {
        arguments: {
            length: 0
        },
        scope2: 'local scope'
    },
    Scope: [AO, [[Scope]]]
}
// 查找到 scope2 的值 返回后函数执行完毕 将函数的执行上下文栈弹出
ECStack = [
    globalContext
];

    # 5.JavaScript this

    ECMAScript的类型分为 语言类型和规范类型

    语言类型是开发者可以直接使用的：undefined null number string boolean object

    规范类型 描述语言底层行为逻辑

    # 1.Reference

    解释 delete typeof 以及 赋值操作的

    构成：

    ​ base value 属性所在的对象或者 environmentRecord 值只能是 undefined ，an object ，a boolean a string a number an environmentRecord

    ​ referenced name：属性的名称

    ​ strict referenced

    栗子：

    var foo = 1;

var fooReference = {
    base : enviromentRecord,
    name:foo,
    strict:false
};
GetValue(fooReference);// 1

    调用 GetValue，返回的将是具体的值，而不再是一个 Reference

    # 2.如何确定this 的值

    1.计算 MemberExpression（属性访问表达式） 的结果赋值给 ref

    2.判断 ref 是不是一个 Reference 类型

    2.1 如果 ref 是 Reference，并且 IsPropertyReference(ref) 是 true, 那么 this 的值为 GetBase(ref)

2.2 如果 ref 是 Reference，并且 base value 值是 Environment Record, 那么this的值为 ImplicitThisValue(ref)

2.3 如果 ref 不是 Reference，那么 this 的值为 undefined

    栗子：

    function foo() {
    console.log(this)
}

foo(); // MemberExpression 是 foo

function foo() {
    return function() {
        console.log(this)
    }
}

foo()(); // MemberExpression 是 foo()

var foo = {
    bar: function () {
        return this;
    }
}

foo.bar(); // MemberExpression 是 foo.bar

// 简单来说 就是 括号前面的内容

    var value = 1;

var foo = {
  value: 2,
  bar: function () {
    return this.value;
  }
}

//示例1
console.log(foo.bar());// 2
//示例2
console.log((foo.bar)());// 2
//示例3
console.log((foo.bar = foo.bar)()); // 1
//示例4
console.log((false || foo.bar)()); // 1
//示例5
console.log((foo.bar, foo.bar)());// 1

//MemberExpression 是 foo.bar, 是 reference 
var reference = {
